# 当前趋势与未来方向

以下判断基于公开论文、技术博客与社区讨论，**非预测性承诺**，仅作架构规划参考。

## 正在发生的收敛

- **多阶段漏斗仍是主流**：生成式方法更多嵌入在召回 / 粗排 / 重排某一环，而非一夜替代全链路。
- **语义 ID + 生成式检索** 与 **双塔 + ANN** 并行探索：前者利于与自回归统一，后者工程成熟；混合架构（多路召回融合）仍常见。
- **LLM 侧重「侧路价值」**：解释、对话、策略与约束检查，比「端到端纯 LLM 打分」更容易满足延迟与可控性。

## 值得关注的方向

1. **更小、更专的推荐模型 + 蒸馏**：在保持吞吐的前提下吸收大模型能力。
2. **对齐与安全**：生成式文案、导购话术与平台合规（医疗、金融、未成年人等）的联合治理。
3. **跨域与跨模态**：图文、短视频、直播切片与商品 SKU 的统一索引与生成式摘要。
4. **评测体系**：除离线 Recall / NDCG 外，长周期用户满意度、对话轮次、转化与负反馈的联合优化。

## 对团队意味着什么

- 算法、工程、产品与法务需要 **更早** 介入同一套「生成边界」设计。
- **观测与回溯**：生成式链路更需要完整的日志与可复现的 Prompt / 工具调用记录。
